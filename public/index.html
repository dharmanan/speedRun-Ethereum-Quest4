üîê Vulnerability Details

The DiceGame contract uses blockhash() combined with contract address and a nonce to generate "random" numbers. However, this is completely predictable because:

  • All blockchain data is public
  • Block hash is known to everyone
  • Contract address is fixed and known
  • Nonce increments predictably
  • keccak256 deterministic function

üí° Solution

The RiggedRoll contract:

  • Calculates the same hash as DiceGame
  • Predicts the roll outcome before calling
  • Only executes rollTheDice() when guaranteed to win (roll 0-5)
  • Includes receive() to accept ETH
  • Includes withdraw() with owner access control
